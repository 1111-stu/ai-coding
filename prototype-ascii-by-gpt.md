# Chatbot 欢迎页 ASCII 原型图

## 1. 设计目标

基于 `style.md` 和 `style-guide.md`，实现一个 DeepSeek 风格的 chatbot 欢迎 UI 页面。当前阶段只对齐页面元素、布局结构和视觉层级，不进入 React/Tailwind 代码实现。

核心方向：

- 浅色、低对比、偏蓝灰的对话产品界面。
- 左侧固定会话侧栏，右侧主区域承载欢迎态。
- 主操作集中在中央输入区，视觉焦点为模式选择和消息输入框。
- 组件使用细边框、轻阴影、胶囊按钮、大圆角输入容器。

## 7. 状态设计

### 7.1 模式按钮

| 状态 | 快速模式 | 专家模式 |
| --- | --- | --- |
| default | active | inactive |
| hover | 使用 `--dsw-alias-button-ghost-active-hover` | 使用 `--dsw-alias-interactive-bg-hover` |
| active | 浅蓝底 + 蓝色边框 + 蓝色文字 | 切换为浅蓝底 + 蓝色边框 |
| focus | 需要可见 focus ring，建议使用主色低透明外环 | 同左 |

### 7.2 输入框

| 状态 | 表现 |
| --- | --- |
| empty | 显示占位文本，发送按钮为弱化状态 |
| focused | 边框增强或轻微阴影增强，背景保持白色 |
| typing | 占位消失，发送按钮可用 |
| loading | 发送按钮替换为停止/加载状态，输入可按产品策略禁用或保留 |
| disabled | 容器透明度降低，按钮不可交互 |

### 7.3 侧栏会话项

| 状态 | 表现 |
| --- | --- |
| default | 透明背景，主文本色 |
| hover | `#f1f3f5` 或 `rgb(var(--ds-rgb-blue-50))` |
| active | `#ebeef2`，可选浅蓝 accent |
| overflow | 单行省略，不换行 |

## 8. 移动端结构建议

移动端不建议常驻 261px 侧栏，应改为抽屉或隐藏式导航。

```text
+---------------------------------------------------+
| Mobile Header                                     |
| [menu]  [logo] deepseek                    [user] |
+---------------------------------------------------+
|                                                   |
|                                                   |
|            [brand icon]                           |
|            使用快速模式开始对话                    |
|                                                   |
|          +------------+ +------------+             |
|          | 快速模式    | | 专家模式    |             |
|          +------------+ +------------+             |
|                                                   |
|  +---------------------------------------------+  |
|  | 给 DeepSeek 发送消息                         |  |
|  |                                             |  |
|  | +----------+ +----------+        [clip] [^] |  |
|  | | 深度思考 | | 智能搜索 |                  |  |
|  | +----------+ +----------+                  |  |
|  +---------------------------------------------+  |
|                                                   |
+---------------------------------------------------+
```

移动端规则：

- Header 高度建议 52-56px。
- Composer 贴近底部或保持欢迎态居中，二选一；如果是欢迎页，建议居中偏下但保留底部安全距离。
- 输入框左右 margin 建议 16px。
- 侧栏入口使用 menu button，点击后打开 drawer。

## 9. 实现前待确认

1. 是否保留左侧完整历史会话侧栏，还是只做欢迎页主区域。
2. 是否需要复刻 DeepSeek logo，还是使用抽象品牌图标占位。
3. 发送按钮空输入时是否禁用。
4. “深度思考 / 智能搜索”是否固定展示，还是作为可配置工具列表。
5. 移动端输入框是否固定在底部。

## 10. 推荐实现优先级

1. AppShell：侧栏 + 主区域。
2. Sidebar：logo、新建对话、会话分组、底部用户。
3. WelcomeMain：标题、模式切换、输入框。
4. Composer 交互状态：focus、typing、disabled、loading。
5. 响应式：桌面侧栏固定，移动端侧栏抽屉化。
